# Contributing to A2A Payment Protocol

Thank you for your interest in contributing to the A2A Payment Protocol specification.

## What This Repo Is

This repository contains a **protocol specification**, not a software library. The reference implementation lives in a separate private repository.

## What We Accept

- **Documentation improvements** — typo fixes, clarifications, better examples
- **Translations** — localised versions of the spec and docs
- **Security reports** — see [SECURITY.md](SECURITY.md) for responsible disclosure
- **Spec feedback** — open an issue to discuss ambiguities, edge cases, or improvements to the protocol

## What We Don't Accept

- **Code contributions** — this repo has no runnable code
- **Alternative implementations** — please publish these in your own repository and let us know
- **Feature requests for the platform** — this repo covers the protocol spec only

## Process

1. **Open an issue** describing the change you'd like to make
2. **Fork** the repository
3. **Create a branch** from `main`
4. **Make your changes** following the style guide below
5. **Open a pull request** referencing the issue

## Style Guide

- Use **ATX-style headings** (`#`, `##`, `###`)
- Use **GitHub Flavoured Markdown (GFM)** tables
- API routes must use the `/pulse-api/` prefix
- Do not reference internal service names, class names, or infrastructure details
- Keep language clear and accessible to implementors outside our organisation

## Code of Conduct

Be respectful, constructive, and professional. We are building an open standard and welcome diverse perspectives.
